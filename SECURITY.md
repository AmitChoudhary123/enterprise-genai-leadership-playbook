# Security Policy

This repository is a public thought-leadership and playbook repository. Do not contribute confidential client information, proprietary architecture, secrets, access tokens, private prompts, regulated data, or internal operating procedures.

## Reporting concerns

Open an issue for non-sensitive concerns. For sensitive concerns, contact the maintainer through the profile contact channel and avoid posting private data publicly.

## Content safety principles

- Use anonymized examples
- Avoid client-identifiable details
- Do not publish secrets or private prompts
- Do not include production credentials, endpoints, or internal system names
- Treat governance examples as reference patterns, not legal advice
