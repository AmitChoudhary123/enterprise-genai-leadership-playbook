# GenAI Operating Model Canvas

This canvas is designed for executive workshops. The output should be operating decisions, not aspirations.

## 1. Design choices

| Operating model choice | Option A | Option B | Option C | Recommendation |
| --- | --- | --- | --- | --- |
| Delivery ownership | Central AI team | Domain squads | Federated product squads | Federated with central standards |
| Platform | Business-unit tools | Enterprise AI platform | Hybrid | Platform for common capabilities, local flexibility for workflow apps |
| Governance | Central review board | Embedded risk partners | Hybrid | Risk tiering plus embedded review |
| Funding | Project-by-project | Product funding | Platform funding | Product/platform mix |
| Talent | Specialist pool | Embedded AI roles | Hybrid | Hybrid with communities of practice |

## 2. Capability map

| Capability | Centralize | Federate | Why |
| --- | --- | --- | --- |
| Security and access standards | Yes | No | Consistency and risk control |
| Model/provider patterns | Yes | Limited | Avoid fragmented vendor decisions |
| Context ownership | No | Yes | Business owns knowledge quality |
| Use-case backlog | No | Yes | Domains own workflow priorities |
| Evaluation standards | Yes | Execute locally | Common gates, domain test cases |
| Adoption/change | No | Yes | Workflow behavior is domain-specific |
| Observability platform | Yes | Consume locally | Shared reliability and cost visibility |

## 3. Leadership operating cadences

| Cadence | Participants | Decisions |
| --- | --- | --- |
| Weekly squad review | Product, engineering, business owner | Delivery, blockers, user feedback |
| Biweekly architecture/risk review | Architect, risk, security, data owner | Context, controls, integration, evaluation |
| Monthly portfolio review | Executives, domain leads, platform | Fund, scale, pause, stop |
| Quarterly capability review | CIO/CDO/CTO, HR, finance | Talent, platform, funding, maturity |

## 4. Maturity assessment

| Maturity signal | Immature | Mature |
| --- | --- | --- |
| Use-case selection | Idea backlog | Value/risk portfolio |
| Context | Document upload | Governed knowledge product |
| Evaluation | Demo review | Release-readiness gates |
| Governance | Late approval | Embedded controls |
| Funding | Experiments | Product/platform investment |
| Adoption | Training | Workflow redesign |

## 5. Workshop output

At the end of the session, capture:

```text
Top 5 funded AI workflows:
Central platform capabilities:
Domain ownership model:
Risk-tier controls:
Evaluation standards:
Governance cadence:
Adoption/change owners:
90-day decisions:
```

The operating model is real only when decision rights, funding, and review cadences are explicit.
