# RAG Readiness Checklist

- Business question is clearly defined
- Source owners are named
- Documents are current and approved
- Metadata supports filtering and access control
- Retrieval quality can be tested
- Citations are required for factual answers
- Evaluation set includes edge cases
- Failure review owner is assigned
- Refresh and retirement rules exist
- Security review covers sensitive content

A RAG system is not ready because documents were indexed. It is ready when evidence can be trusted and failures can be managed.
