# Use-Case Prioritization Framework

A strong GenAI portfolio is not built by collecting ideas. It is built by forcing tradeoffs.

## Scoring dimensions

| Dimension | Low score | High score |
| --- | --- | --- |
| Business value | Convenience improvement | Material revenue, cost, risk, cycle-time, or experience impact |
| Workflow readiness | Process is unclear or highly variable | Workflow is known, measurable, and owned |
| Context readiness | Knowledge is scattered, stale, or politically owned | Sources are approved, current, and accountable |
| Evaluation readiness | Success is subjective | Metrics and test cases can be defined |
| Risk manageability | Controls are unclear or heavy | Risk tier and controls are practical |
| Adoption feasibility | Requires major behavior change | Fits into natural work patterns |

## Decision rule

| Score pattern | Recommendation |
| --- | --- |
| High value, high readiness | Fund as priority pilot |
| High value, low context readiness | Fix knowledge/process foundation first |
| Medium value, high readiness | Use as capability-building quick win |
| Low value, high complexity | Defer |
| High risk, unclear controls | Do not automate action; start with decision support |

## Leadership anti-pattern

Do not let executive enthusiasm override context and evaluation readiness. That is how expensive pilots become reputational risk.
